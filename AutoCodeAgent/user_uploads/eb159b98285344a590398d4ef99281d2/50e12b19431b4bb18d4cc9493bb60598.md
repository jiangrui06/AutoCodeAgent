# 多模型多 Agent Code Agent 任务清单

> 产品定位：从零开发；自部署模型；自动选择合适模型；主 Agent 调度多个专业子 Agent 完成代码研发任务。

## Phase 0：定位与可行性

- [ ] Task 1：冻结产品边界与关键 ADR
- [ ] Task 2：验证 Dense 与 MoE 自部署推理链路
- [ ] Task 3：建立模型能力基准与初始角色映射
- [ ] Checkpoint A：模型能力、部署容量和产品范围通过评审

## Phase 1：模型与编排基础设施

- [ ] Task 4：初始化服务工程与质量门禁
- [ ] Task 5：定义核心领域契约与事件协议
- [ ] Task 6：实现模型注册中心与统一推理客户端
- [ ] Task 7：实现持久事件、任务和产物存储
- [ ] Checkpoint B：可注册模型并恢复静态 Task DAG

## Phase 2：智能路由与 Agent Runtime

- [ ] Task 8：实现可解释的模型路由器
- [ ] Task 9：实现单 Agent Runtime 与结构化产物
- [ ] Task 10：实现 Task DAG 校验与调度器
- [ ] Task 11：实现主 Agent 与专业子 Agent 模板
- [ ] Checkpoint C：Dense 规划、MoE 开发、多子 Agent 编排跑通

## Phase 3：代码研发闭环

- [ ] Task 12：实现仓库分析与上下文产物层
- [ ] Task 13：实现隔离工作区与安全工具网关
- [ ] Task 14：实现补丁集成、冲突处理与回滚
- [ ] Task 15：实现测试、独立审查与修复循环
- [ ] Checkpoint D：真实仓库端到端研发任务通过

## Phase 4：产品化与试运行

- [ ] Task 16：交付 REST API 与验证 CLI
- [ ] Task 17：建立端到端评测与路由效果评估
- [ ] Task 18：补齐可观测性与运维控制面
- [ ] Task 19：受控试运行与发布门
- [ ] Checkpoint E：MVP 发布就绪

## P0 决策清单

- [ ] 确认两个候选模型的准确权重、许可证和上下文规格
- [ ] 确认 GPU 型号、数量、显存和网络拓扑
- [ ] 确认 vLLM、SGLang 或内部推理平台
- [ ] 确认内部单租户或企业多租户定位
- [ ] 确认首发语言、仓库规模和构建系统
- [ ] 确认容器、虚拟机或现有安全沙箱
- [ ] 确认最大子 Agent 并发和层级
- [ ] 确认自动执行与人工审批边界

## MVP 关键验证

- [ ] Dense 模型完成需求理解、仓库分析和 Task DAG 规划
- [ ] MoE 模型完成边界明确的编码与测试任务
- [ ] 主 Agent 可调度至少两个独立子 Agent
- [ ] 并行写任务使用隔离 worktree，不覆盖用户或彼此改动
- [ ] 集成、测试和独立 Review 全部通过后才声明成功
- [ ] 模型故障、任务失败、冲突和取消能够安全收敛
- [ ] 智能路由相对固定单模型基线具有可量化收益

完整架构、任务验收、风险和排期见 `tasks/plan.md`。
